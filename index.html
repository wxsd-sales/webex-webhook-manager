Webex Webhook Manager

Connect your bot

Paste a Webex bot access token to load its webhooks. The token is used only in your browser to call the Webex APIs and is never stored or sent anywhere else — it is forgotten as soon as you disconnect or reload the page.

Create bots and grab their tokens at developer.webex.com/my-apps.

Create webhooks

Point your bot at an AWS Lambda function (or any HTTPS endpoint). Select the resources your bot needs and create all of the matching webhooks in one go. For a typical card‑based bot you want messages and attachmentActions on the created event.

Your AWS Lambda Function URL or API Gateway endpoint. Must be HTTPS and publicly reachable.
Each webhook is named “prefix – resource”.
Webex signs each payload so your Lambda can verify it.

Generate Test Payloads

Build signed AWS Lambda test events that mirror what Webex delivers to your bot. Pick the resources you want, tweak the message or card inputs, then generate one ready‑to‑paste test event per resource — each with a valid x-spark-signature header derived from your shared secret. Everything is computed locally in your browser.

Webex signs each payload with HMAC‑SHA1(body, secret) and sends it as the X‑Spark‑Signature header. Leave blank to generate an unsigned payload.
